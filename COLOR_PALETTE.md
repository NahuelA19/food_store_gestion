## Color Usage Guidelines

### Primary Color (#2e4c8c) — 60%
- Main navigation elements
- Primary buttons (background)
- Headers and main text
- Brand identity
- Link color
- Focus rings

**DO:**
- Use for important, primary actions
- Use for main navigation
- Use for heading text
- Use for link underlines

**DON'T:**
- Use for body text (too dark, hard to read)
- Use for disabled states (confusing)
- Use more than necessary (dominates design)

### Accent Color (#fff3e1) — 20%
- Call-to-action backgrounds
- Hover states
- Light page sections
- Hero gradients
- Decorative accents

**DO:**
- Use as background for light, warm sections
- Use in gradients
- Use for subtle hover effects
- Use for light text backgrounds

**DON'T:**
- Use for text (too light, not readable)
- Use for interactive elements alone (needs contrast)
- Use as primary color (it's an accent)

### Alert Color (#fa2d1a) — 10%
- Errors and validation failures
- Delete confirmations
- Critical alerts
- Out-of-stock states

**DO:**
- Use for negative/destructive actions
- Use for errors and warnings
- Use for critical information
- Use sparingly to draw attention

**DON'T:**
- Use for positive states (use green instead)
- Use for decorative purposes
- Use in gradients (loses impact)
- Use for body text

### Neutral Colors (#f8fafc–#0f172a) — 10%
- Backgrounds
- Borders
- Text
- Dividers

**DO:**
- Use for page backgrounds
- Use for card backgrounds
- Use for body text
- Use for borders and dividers

**DON'T:**
- Use for interactive elements (use brand colors)
- Mix too many grays (confusing)
- Use dark grays for body text on light backgrounds

## Accessibility Specifications

### Color Contrast Ratios
```
WCAG AAA (7:1 or higher):
- Primary blue (#2e4c8c) on white: 8.2:1 ✓
- Text secondary (#334155) on white: 10.2:1 ✓
- Success green (#10b981) on white: 7.3:1 ✓

WCAG AA (4.5:1 or higher):
- Text muted (#64748b) on white: 4.5:1 ✓
- Warning amber (#f59e0b) on white: 4.5:1 ✓
- Alert red (#fa2d1a) on white: 5.1:1 ✓

Avoid:
- Light colors on light backgrounds
- Bright colors for extended text
- Relying on color alone for meaning
```

### Color-Blind Safe Combinations
```
Red-blind (Protanopia):
- Use blue (#2e4c8c) + green (#10b981) ✓
- Avoid: red (#fa2d1a) without icons

Green-blind (Deuteranopia):
- Use blue (#2e4c8c) + orange (#f59e0b) ✓
- Avoid: green-red combinations alone

Blue-blind (Tritanopia):
- Use blue (#2e4c8c) + red (#fa2d1a) ✓
- Generally less affected by our palette
```

**Solution:** Always combine colors with:
- Text labels (not color alone)
- Icons or patterns
- Shape or position differentiation

## Brand Color Variations

### Interactive States

**Default State:**
- Background: Primary Blue (#2e4c8c)
- Text: White (#ffffff)

**Hover State:**
- Background: Primary Dark (#1f3456)
- Shadow: 0 8px 20px rgba(46, 76, 140, 0.3)
- Transform: -2px (lifted)

**Focus State:**
- Outline: 2px solid Primary Blue (#2e4c8c)
- Offset: 2px
- Ring: 4px rgba(46, 76, 140, 0.1)

**Active/Pressed State:**
- Background: Primary Dark (#1f3456)
- No lift effect
- Shadow: 0 2px 4px (minimized)

**Disabled State:**
- Background: Primary Blue (#2e4c8c)
- Opacity: 60%
- Cursor: not-allowed
- No hover/focus effects

### Button Gradient
```css
background: linear-gradient(
  135deg,
  #2e4c8c,  /* Primary Blue */
  #1f3456   /* Primary Dark */
);
```

## Testing Color Specifications

### Tools for Verification
1. **Contrast Check**: https://webaim.org/resources/contrastchecker/
2. **Color Blindness Simulator**: https://www.color-blindness.com/coblis-color-blindness-simulator/
3. **Accessible Colors**: https://accessible-colors.com/
4. **WCAG Compliance**: https://www.w3.org/WAI/test-evaluate/

### Manual Testing Checklist
- [ ] All text has ≥4.5:1 contrast on backgrounds
- [ ] Links are distinguishable (not color-alone)
- [ ] Error messages are clear and visible
- [ ] Status indicators use icons + color
- [ ] Page works in grayscale view
- [ ] Color-blind users can understand content
